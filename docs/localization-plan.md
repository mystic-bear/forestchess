# Forest Chess 한글/영문 전환 계획

## 1. 목표

- 시작 화면에서 `한국어 / English`를 선택할 수 있게 한다.
- 시작 화면, 커스텀 세팅, 대국 셸, 상태 메시지, 코치 힌트까지 같은 번역 계층으로 관리한다.
- 문자열 하드코딩을 줄여 이후 일본어 등 추가 언어 확장 비용을 낮춘다.

## 2. 현재 문제

- UI 문구가 `index.html`, `js/ui.js`, `js/game.js`, `worker/coach-dispatcher.js`에 흩어져 있다.
- 동물 말 이름, 프리셋 설명, 버튼 문구, 오류 메시지가 서로 다른 방식으로 관리된다.
- 언어를 바꾸려면 파일 여러 곳을 동시에 수정해야 해서 회귀 위험이 크다.

## 3. 적용 원칙

- 공개 언어는 `ko`, `en` 두 개만 먼저 지원한다.
- 시작 전 UI에서 언어를 고르고, 선택값은 로컬 저장으로 유지한다.
- 구조화 데이터는 `shared/constants.js`에서 다국어 값을 함께 가진다.
- 일반 UI 문장은 `shared/i18n.js`의 키 기반 번역으로 관리한다.
- 체스 규칙/엔진 로직은 언어와 분리하고, 화면에 노출되는 문장만 번역 계층을 탄다.

## 4. 구현 구조

### 4-1. 번역 계층

- `shared/i18n.js`
  - 지원 언어 목록
  - 기본 언어
  - 번역 키 사전
  - `translateUi(language, key, params)` 제공

### 4-2. 구조화 상수

- `shared/constants.js`
  - 플레이어명
  - AI 설명
  - 프리셋 문구
  - 동물/체스 말 이름
  - 보드 방향 라벨
  - 힌트 단계 라벨

### 4-3. 언어 상태

- `js/game.js`
  - 현재 언어 상태 보관
  - `localStorage`에 저장/복원
  - worker로 `language` 전달

### 4-4. 렌더링

- `js/ui.js`
  - 시작 전 언어 선택 버튼 렌더링
  - `data-i18n` 기반 정적 텍스트 반영
  - 동적 패널/버튼/상태문구 현지화

### 4-5. 워커/코치

- `worker/ai-worker.js`
  - `gameState.language` 전달
- `worker/coach-dispatcher.js`
  - 힌트 카드 문장과 대안 수 설명도 선택 언어에 맞춰 생성

## 5. 용어 기준

- 기본 한국어 제품 톤은 쉬운 설명을 우선한다.
- 기물 표기는 기본값을 `동물 + 체스`로 둔다.
  - 예: `고양이 (폰)`, `곰 (킹)`
- 주요 체스 용어는 아래 기준으로 유지한다.
  - Check: `체크`
  - Checkmate: `체크메이트`
  - Stalemate: `스테일메이트`
  - Draw: `무승부`

## 6. 단계별 적용 범위

### Phase A. 시작 전 UI

- 시작 화면
- 빠른 시작 프리셋
- 커스텀 세팅
- 언어 선택 토글

### Phase B. 대국 셸

- 상단 바
- 좌/우 사이드 패널
- 버튼
- 상태 칩
- 프로모션/결과 모달

### Phase C. 코치/오류

- 코치 힌트 카드
- 엔진 로드 실패 메시지
- 토스트 메시지
- 보드 정보 패널

### Phase D. 후속 확장

- 복기 모드
- 저장/불러오기 문구
- 향후 퍼즐/레슨 모드

## 7. 수용 기준

- 시작 화면에서 언어를 바꾸면 즉시 텍스트가 전환된다.
- 페이지를 다시 열어도 마지막 언어 선택이 유지된다.
- 말 툴팁과 범례도 선택 언어 기준으로 바뀐다.
- AI/코치 힌트 카드도 선택 언어 기준으로 생성된다.
- 영어/한국어 어느 쪽에서도 레이아웃이 깨지지 않는다.

## 8. 리스크와 대응

- 문자열 누락
  - 번역 키 누락 시 기본 언어로 폴백
- 워커와 메인 스레드 언어 불일치
  - `buildGameStateForAI()`에 `language`를 명시 포함
- 긴 한국어 문장으로 인한 UI 깨짐
  - 모바일/데스크톱 QA에서 줄바꿈 확인

- 힌트 난도에 따라 어린이용 문장/일반 문장 톤 분리
- 번역 키 lint 또는 누락 검사 스크립트 추가
